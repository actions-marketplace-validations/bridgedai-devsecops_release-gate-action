# BridgedAI Release Gate (`bridgedai-devsecops/release-gate-action`)

## What this action does

Enforces BridgedAI policy outcomes using `observe`, `warn`, or `enforce` modes and always writes a job summary when enabled.

## Why BridgedAI exists

CI “green” is not sufficient; policy outcomes must be enforced consistently.

## Quick start

See `examples/basic.yml`.

## Enterprise setup

Use environment protections around workflows that run `enforce` mode.
